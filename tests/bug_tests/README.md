Bugs Section (README)
Overview
NovaScript, like any programming language, is not 100% bug-free. This section is meant to track runtime errors, script issues, and syntax problems as they are discovered. Each bug entry should include the file path, error type, and details, and should be removed once fixed.
How to Use This Section
Add New Bugs:
Whenever a new bug is discovered during testing or runtime, add a new entry with:
File: Path to the script
Type: Error type (e.g., TypeError, SyntaxError, NovaError)
Description: Brief explanation of the issue
Location: Line and column numbers if available
Notes: Any additional context or reproduction steps
Update Bugs:
If a bug is fixed, remove it from this section to keep the list current.
Purpose:
Provides developers a quick reference for known issues.
Helps in testing and avoiding repeated errors.
Serves as a community tracking tool for NovaScript bugs.
This version keeps the section clean, reusable, and continuously updateable without listing specific bugs.
